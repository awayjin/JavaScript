1. 项目中存在的问题, 对应的优化

  (1) 构建速度慢，3分钟左右，怎样优化？
      - 速度分析, speed-measure-webpack-plugin
      - 使用 yarn 而不是 npm,
      - webpack 优化生成 stats 文件

  (2) 技术人员水平参差不齐、代码风格不一，工具保证(eslint), 需要前端团队规范
      - 前端团队规范
      - Code Review建立一种review的文化

  (3) 某些页面代码过长，可读性差、不好维护，没有分模块

  (4) 背景不一样，不是所有人员都能使用好 git, 编写好 git 常用命令

  (5) 随项目的迭代，模块越来越多，路由、请求资源、样式、图片都需要按模块来分

  (6) 一次数据加载太多(1000多条)，导致操作明显卡顿， element-ui 表格渲染太慢
